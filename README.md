# Cервис сбора и агрегирования статистики видео

Проект представляет собой сервис для автоматического сбора и агрегирования статистики просмотров видео с различных платформ. Взаимодействие с пользователем осуществляется через Telegram-бота.

## Архитектура

Система построена на основе микросервисной архитектуры и включает в себя три основных компонента, работающих в Docker-контейнерах:

1.  **База данных (`db`)**
    *   **Технология:** PostgreSQL 14.
    *   **Роль:** Центральное хранилище данных. Используется для обмена информацией между телеграм-ботом и бэкенд-сервисом, что обеспечивает их слабую связанность.
    *   **Схема:** Структура базы данных описана в файле `schema.sql`.

2.  **Бэкенд-сервис (`backend`)**
    *   **Технология:** java 17, java.net.http, java.sql (без использования Spring).
    *   **Роль:** Основной обработчик данных. Это фоновый сервис, который периодически выполняет следующие действия:
        1.  Запрашивает из базы данных список всех добавленных URL.
        2.  Обращается к API соответствующей видео-платформы (YouTube, RuTube, Vimeo) для получения актуального количества просмотров.
        3.  Сохраняет обновленную статистику обратно в базу данных.
    *   **Поддерживаемые платформы:** YouTube, RuTube, Vimeo.

3.  **Телеграм-бот (`bot`)**
    *   **Технология:** Python с использованием библиотеки `aiogram`.
    *   **Роль:** Пользовательский интерфейс. Бот обрабатывает команды пользователя, позволяя выполнять следующие действия:
        *   Добавлять новые ссылки на видео для отслеживания.
        *   Просматривать общую и детальную статистику по добавленным видео.
        *   Удалять видео из списка отслеживания.
    *   Бот напрямую взаимодействует с базой данных для добавления URL и получения уже обработанной статистики.

## Технологический стек

*   **Бэкенд:** Java 17, Maven
*   **Телеграм-бот:** Python 3.10, aiogram
*   **База данных:** PostgreSQL 14
*   **Оркестрация:** Docker, Docker Compose

## Запуск и развертывание

Для запуска проекта необходимо выполнить следующие шаги:

**1. Подготовка конфигурационного файла**

Проверить переменные в файле `.env`

**2. Сборка и запуск контейнеров производятся командой ./run.sh**

Содержимое run.sh:

```
#!/bin/bash
set -e

docker compose down

docker compose up -d --build

docker ps --filter "name=hakaton-"
```

Он соберет образы для бэкенда и бота, запустит все три контейнера "hakaton-..." в фоновом режиме и применит схему schema.sql базы данных.

**3. Остановка и полная очистка содержимого контейнеров скриптом ./stop.sh**

Содержимое stop.sh:

```
#!/bin/bash
set -e

# очистка video_stats, urls, users_urls и перезапуск инкремента id
#docker exec -i hakaton-db psql -U hakaton_user -d hakaton_db -c "TRUNCATE video_stats, urls, users_urls RESTART IDENTITY CASCADE;"
docker compose down -v
```
Бот и бэкенд после запуска пишут логи соответственно в файлы: bot_log, bot_backend_log в текущей директории

```

## Использование телеграм-бота

После успешного запуска найдите вашего бота в Telegram и начните взаимодействие с ним.

**Основные команды и функции:**

*   `/start` - Инициализация бота, регистрация пользователя и отображение главного меню.

**Кнопки главного меню:**

*   **Мой контент**: Показывает общую статистику (количество ссылок и суммарное число просмотров) и выводит список добавленных видео в виде кнопок для просмотра детальной информации.
*   **Управление**: Открывает меню для добавления или удаления видео.
    *   **Добавить видео**: Запрашивает у пользователя ссылку на видео. После отправки корректной ссылки, она добавляется в систему для отслеживания.
    *   **Удалить видео**: Позволяет выбрать и удалить ранее добавленное видео из списка отслеживания.
*   **Поддержать проект**: Функция для донатов.
